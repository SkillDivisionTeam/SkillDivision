## 2. Необходимость проекта

### 2.1. Обоснование необходимости

IT сообщество видит проблему в современном подходе к meet-up. Отсутствие прямых каналов связи после мероприятия. Скучные бумажные анкеты для тестирования навыков. Низкая эффективность HR-специалистов, из-за невозможности провести беседу со всеми потенциальными кандидатами.

### 2.2. Видение проекта

В течении полугода Skill Division пройдет путь от простого Telegram-бота до аналитической платформы с несколькими интерфейсами, собственным ИИ и гибкой кастомизации под любой вид мероприятий.

### 2.3. Анализ выгод

**Выгоды для it-сообщества:**

- Интерактив на мероприятии доступный каждому
- Легко заявить о себе окружающим
- Доступность информации о мероприятии
- Доступность информации об участниках

**Выгоды для HR-специалистов:**

- Отчет по компетенциям участников
- Сегментация участников по уровню знаний той или иной области
- Простой способ связаться с нужным участником мероприятия

**Выгоды для организаторов:**

- Узнаваемость бренда
- Отчет об активности участников
- Упрощенная коммуникация с участниками
- Удобный интерфейс с информацией о мероприятии

## 3. Концепция решения

Система «Skill division» представляет собой комбинацию веб-приложения для организаторов и HR-специалистов и Telegram-бота для участников мероприятий. Система состоит из двух интерфейсов с частичным дублированием функционала:

- Telegram-бот для всех пользователей: позволяет регистрироваться на мероприятии, участвовать в квизе(одиночном и дуэлях), получать информацию о мероприятии, получать таблицу лидеров квиза.
- Веб-приложение для организаторов и HR-специалистов: обеспечивает анализ квиза, формирует подробный отчет об участниках квиза, хранит информацию о мероприятиях, позволяет кастомизировать бота под мероприятие с помощью ИИ.

Решение предполагает централизованное хранение данных, защищенный доступ и экспорт данных в CSV для HR-специалистов. Система рассчитана на мероприятия до 100 участников (с возможностью расширения в будущем).

### 3.1. Цели и Задачи

#### 3.1.1. Цели проекта

**Повышение вовлеченности участников IT-митапов и встреч:**

- Создание интерактивной платформы на базе Telegram-бота, где участники смогут участвовать в квизах, связанных с темами мероприятий, для повышения интереса и активности.

**Сбор и анализ данных для HR и организаторов:**

- Обеспечение сбора статистики по знаниям и навыкам участников через квизы для поддержки процесса найма и оценки качества мероприятий.

**Автоматизация подготовки контента:**

- Разработка системы с использованием ИИ для автоматической генерации вопросов квиза на основе указанных тем и уровней сложности, упрощающей работу организаторов.

**Создание удобного инструмента для организаторов и HR:**

- Предоставление веб-интерфейса для управления мероприятиями, создания квизов, мониторинга статистики и поиска талантов, с доступом через индивидуальные логины и пароли.

**Стимулирование профессионального роста участников:**

- Оценка навыков участников через квизы и предоставление рекомендаций для развития, а также интеграция с профильными платформами для карьерных возможностей.

#### 3.1.2. Задачи проекта

**Разработка Telegram-бота:**

- Реализация бота для проведения квизов, регистрации участников и предоставления базовой статистики в реальном времени.

**Создание веб-интерфейса:**

- Разработка дашборда для организаторов (создание мероприятий, генерация вопросов, мониторинг) и HR (анализ талантов, экспорт данных в CSV/PDF).

**Интеграция ИИ-системы:**

- Интеграция ИИ для автоматической генерации вопросов квиза с возможностью редактирования и валидации.

**Обеспечение аналитических функций:**

- Реализация модуля для построения Skill Division, анализа данных участников и формирования отчётов с инсайтами для HR и организаторов.

**Обеспечение безопасности и соответствия нормам:**

- Внедрение механизмов защиты данных (согласие участников, шифрование).

**Тестирование и запуск MVP:**

- Проведение пилотного запуска на 1-2 митапах с ограниченным числом пользователей для сбора обратной связи и доработки платформы.

**Масштабирование и развитие:**

- Расширение функционала (мобильное приложение, мультиязычность) на основе обратной связи и рыночных потребностей.

### 3.2. Предположения и Ограничения

**Предположения:**

- Все участники мероприятий имеют доступ к интернету, Telegram и базовые навыки работы с веб-приложениями
- Синхронизация данных между Telegram-ботом и веб-интерфейсом будет происходить в реальном времени с задержкой не более 5 секунд.
- Минимум 50% участников мероприятия согласятся на обработку данных для аналитики и экспорта профилей.
- ИИ-модели (например, Yandex GPT или GigaChat) смогут генерировать качественные вопросы с точностью не менее 80% при правильной настройке промптов.
- Организаторы митапов будут готовы предоставлять темы и данные для квизов, а HR — использовать аналитику для поиска талантов.
- Пилотный запуск на 1-2 мероприятиях привлечет не менее 50 участников и 5-10 организаторов/HR для тестирования.
- Проект разворачивается и тестируется в контейнерной среде (Docker), что гарантирует воспроизводимость конфигурации независимо от операционной системы участников команды.

**Ограничения:**

- Система разрабатывается в ограниченные сроки (6 месяцев)
- Пользовательский интерфейс должен быть интуитивно понятным, без необходимости длительного обучения
- Система должна корректно работать в браузере Chrome
- Поддержка мобильных устройств обеспечивается через Telegram-бот для исполнителей

### 3.3. Анализ использования

#### 3.3.1. Пользователи

Система предусматривает три роли: Организаторы, HR-специалисты, участники мероприятий. Участники мероприятий регистрируются через Telegram-бот, Организаторы и HR-специалисты — через веб-интерфейс. Доступ Организаторов и HR-специалистов регулируется разработчиками (через админ панель). Пользователь может иметь только одну роль.  

**Организаторы:**

- Создают и настраивают мероприятия, добавляют информацию от них
- Кастомизируют Telegram-бота под кокретное мероприятие с помощью ИИ
- Просматривают дашборд мероприятия с активностью участников
- Формируют и просматривают отчеты об активности участников
- Видят подробную статистику об участниках

**HR-специалисты:**

- Получают отчеты по компетенциям участников мероприятий
- Видят сегментированную статистику об участниках по уровню знаний
- Используют простой способ связаться с нужным участником мероприятия через систему
- Экспортируют данные об участниках в CSV для дальнейшей работы

**Участники мероприятий:**

- Регистрируются на мероприятия через Telegram-бота
- Получают информацию о мероприятии через боты
- Участвую в квизах (одиночных и дуэлях)
- Легко заявляют о своих навыках и опыте IT-сообществу

### 3.4. Требования

#### 3.4.1. Требования пользователей

**Исполнители должны иметь возможность:**

- Зарегистрироваться на мероприятие через Telegram-бота
- Просматривать информацию о мероприятии (программа, спикеры и т.д)
- Участвовать в одиночных квизах и дуэлях
- Просматривать свой прогресс и рейтинг в таблице лидеров
- Давать согласие на обработку своих данных для аналитики

**Организаторы должны иметь возможность:**

- Создавать мероприятия и управлять информацией о них в веб-интерфейсе
- Кастомизировать бота для мероприятия (название, приветствие, темы квизов)
- Использовать ИИ для автоматической генерации вопросов для квизов
- Просматривать дашборд с активностью участников в реальном времени
- Формировать отчеты об участии и активности на мероприятии

**HR-специалисты должны иметь возможность:**

- Получать детальную аналитику по навыкам
- Сегментировать участников по уровню знаний в разных областях
- Находить и получать контактные данные подходящих кандидатов
- Экспортировать данных об участниках (отчеты, профили) в CSV-формат

#### 3.4.2. Системные требования

**Функциональные требования:**

- Поддержка регистрации, аутентификации и авторизации пользователей с различными ролями.
- Учет и хранение результатов Баттлов.
- Хранение данных о пользователях, мероприятиях и отчетах в базе данных.
- Наличие экспорта данных в CSV для обмена с внешними системами.

**Нефункциональные требования:**

- Доступность системы через Telegram-бот (для участников) и стандартные веб-браузеры (для организаторов/HR).
- Время отклика системы при нормальной нагрузке не более 5 секунд.

## 4. Рамки

### 4.1. Функциональность решения

Данная функциональность направлена на достижение видения проекта: создание интерактивной аналитической платформы для интеллектуальных соревнований на мероприятиях с автоматической сегментацией талантов и генерацией HR-отчётов.

1. Регистрация и участие в мероприятии через Telegram-бот
    1. **Возможность: вовлечение участников в игру с первого касания**
    2. **Функции:**
        1. Быстрая регистрация по Telegram-аккаунту
        2. Вступительное повествование (история мероприятия)
        3. Прохождение фильтрующих вопросов (3–5 этапов)
        4. Автоматическое распределение по интересам и уровню знаний
        5. Участие в дуэлях (1-на-1) с живым оппонентом
        6. Получение очков за правильные ответы
        7. Просмотр таблицы лидеров в реальном времени
        8. Просмотр карточки участника (навыки, уровень, контакты)
2. Веб-приложение для организаторов и HR-специалистов
    1. **Возможность: полный контроль над мероприятием и данными**
    2. **Функции:**
        1. Авторизация по ролям (организатор / HR / админ)
        2. Создание мероприятия: название, дата, описание, логотип
        3. Кастомизация бота через ИИ: генерация вопросов по теме, тону, сложности
        4. Настройка фильтрующих вопросов и весов для сегментации
        5. Мониторинг активности в реальном времени (онлайн-участники, дуэли)
        6. Формирование отчёта по компетенциям (Excel/CSV)
        7. Сегментация участников: beginner / intermediate / advanced
        8. Фильтрация по навыкам, интересам, активности
        9. Массовая и персональная рассылка через Telegram
3. Аналитика и отчётность
    1. **Возможность: превращение игровых данных в ценность для HR**
    2. **Функции:**
        1. Автоматический расчет рейтинга по формуле: Рейтинг = (Правильные ответы × Сложность вопроса) + Бонус за скорость
        2. Генерация топ-10 / топ-50 по направлениям
        3. Сравнение эффективности фильтрующих этапов
        4. Экспорт всех данных в CSV с метками (уровень, навыки, контакты)
4. Уведомления и коммуникация
    1. **Возможность: поддержание вовлечённости и HR-контактов**
    2. **Функции:**
        1. Массовая рассылка: начало дуэли, новые лидеры, завершение
        2. Персональные уведомления: "Вам бросил вызов @user", "Вы в топ-10!"
        3. Прямой переход из бота в чат с участником (с согласия)

### 4.2. За рамками решения

Следующая функциональность была исключена из рамок текущего проекта для соблюдения сроков (3–6 месяцев) и фокуса на MVP.

| Возможность/функциональность       | Причина исключения                                                                                  |
| ---------------------------------- | --------------------------------------------------------------------------------------------------- |
| Мобильное приложение (iOS/Android) | **Стоимость/Сроки.** На первом этапе используется Telegram — 100% покрытие аудитории без установки. |
| ИИ генерация вопросов              | **Сроки/Сложность.** Добавлять свою LLM оставлено на будущее.                                       |

### 4.3. Критерии одобрения решения

Решение будет считаться готовым и принятым Заказчиком при успешном выполнении следующих условий:

- Реализован телеграмм-бот с функциями: Просмотр информации о мероприятии, Просмотр информации о проекте Skill Division, Баттл, просмотр карточки участника, просмотр таблицы лидеров.

- Реализован web-интерфейс с функциями: Авторизации, Создания мероприятия, просмотра статистики мероприятия, просмотра участников.

## 5. Стратегии дизайна решения

### 5.1. Стратегия архитектурного дизайна

Архитектура системы «Skill Division» основана на **трехкомпонентной модели (трехуровневой архитектуре)**, включающей уровень представления, уровень логики приложения и уровень данных.

**Основные компоненты системы:**

1. **Telegram-бот для участников**
    - Отвечает за взаимодействие с пользователями-участниками.
    - Позволяет играть в Баттл, просматривать информацию о проекте Skill Division и мероприятиях, посмотреть карточку участника, смотреть таблицу лидеров.
    - Обменивается данными с сервером через REST API.
2. **Веб-приложение для Организаторов и HR**
    - Предоставляет интерфейс для управления проектами, задачами и пользователями.
    - Позволяет анализировать статистику, выгружать отчеты, создавать и редактировать мероприятия.
    - Работает через браузер и подключается к серверу через REST API.
3. **Серверная часть (Backend)**
    - Реализует бизнес-логику: учёт задач, расчёт показателей производительности, авторизацию пользователей, управление проектами.
    - Является общим центром для Telegram-бота и веб-приложения.
    - Содержит REST API для всех операций.
4. **База данных**
    - Хранит сведения о пользователях, мероприятиях, временных записях и отчетах.
    - Центральный узел хранения данных с обеспечением целостности и безопасности.

Для обеспечения независимости от окружения и упрощения развертывания все основные компоненты системы (backend, база данных, Telegram-бот и frontend) будут контейнеризованы с использованием **Docker**.  

**Взаимодействие -компонентов:**

![Диаграмма компонентов](./images/Component%20diagram.png)

**Архитектурный паттерн:**

- В основе лежит **MVC (Model–View–Controller)**, реализованный средствами Django.
- REST API построен на **паттерне “клиент–сервер”** с четким разделением ответственности.
- Telegram-бот использует архитектурный подход **“thin client”** — минимальная логика на стороне клиента, вся бизнес-логика на сервере.

**Преимущества выбранной архитектуры:**

- Простота разработки и сопровождения.
- Возможность независимой доработки бота и веб-интерфейса.
- Расширяемость — в будущем можно добавить мобильное приложение или интеграцию с CRM без изменения ядра системы.

### 5.2. Стратегия технологического дизайна

| **Компонент**                      | **Технология / Средство**                               | **Обоснование выбора**                                                                                                                             |

## 6. Инфраструктурные решения

Для упрощения разработки и развёртывания система “Skill Division” использует контейнеризацию с помощью Docker.

Основные преимущества подхода:

- единая среда для всех членов команды;
- быстрое развертывание приложения одной командой (docker compose up);
- независимость от локальных конфигураций ОС;
- возможность масштабирования и интеграции с облачными платформами (Yandex Cloud, Render).

Инфраструктура проекта включает четыре контейнера:

1. Backend (Django + DRF) — основная логика и REST API
2. Database (PostgreSQL) — централизованное хранилище данных
3. Frontend (React) — интерфейс для HR и организаторов
4. Telegram Bot (python-telegram-bot) — коммуникация с участниками
